Завдання 

Ви завершили роботу над завданням для клієнта і у вас є робочий код, який був створений без використання Git.
Клієнт створив порожній репозиторій git@example.com:example/test.git і надав вам доступ до нього.

 Опишіть ваші подальші дії, щоб ваш код з'явився у гілці master у репозиторії git@example.com:example/test.git.

 Рішення

Для того щоб код з'явився у гілці master у репозиторії git@example.com:example/test.git треба зробити такі дії

1.Відкрити термінал і перейти до каталогу де знаходиться проект(наприклад для переходу використати команду
 cd Робочій стіл/проекти)

2.Ініціалізірувати проект на локальний Git-репозиторій за допомогою команди
 git init

3.Додати усі файли в індекс Git за допомогою команди
 git add . (крапка додає усі файли які в каталозі)

4.Зафіксувати усі зміни у вашему проекту зробивши комміт
 git commit -m 'Готовий проект'

5.Тепер треба налаштувати віддалений репозиторій git@example.com:example/test.git для цього нам потрібно використати команду
git remote add origin git@example.com:example/test.git

6.Останній крок завантажити файли у віддалений репозиторій за допомогою
 git push -u origin master ця команда надішле код у віддалений репозиторій
